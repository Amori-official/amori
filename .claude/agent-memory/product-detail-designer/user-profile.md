The user is a solo founder/designer building Amori — a Korean baby fabric brand e-commerce store. They are technical (comfortable with Next.js, TypeScript, Tailwind) and have a strong visual/brand sensibility. They provide clear briefs with existing code context and expect complete, working file edits rather than proposals or drafts for approval.

They communicate the task in Korean and expect deliverables to be in Korean where visible copy is concerned, with English for code identifiers.

They provide reference URLs (Naver smartstore) as content source — treat these as the authoritative product specification even if the URL itself cannot be fetched.
